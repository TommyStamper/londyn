# Zasady współpracy - przewodnik po Londynie
Aby pracować w tym repo, trzeba być ogarniętym.

## Zgłaszanie błędów
Tomek nie robi błędów!!!1

## Dodawanie nowych gałęzi
Każda zmiana wymaga stworzenia nowego brancha
Do nazywania gałęzi używamy kebab-case:
-fix-
-feature-
-bugfix-

## Commitowanie
Commit message w oparciu o conventional commit https://www.conventionalcommits.org/en/v1.0.0/

## Pull Requesty
Pull requesty wymagają review przez Tomka
Przetestuj kod zanim stworzysz PR
Opisz zawartość PR

## Styl kodu
Używaj czystego kodu i komentarzy tam gdzie trzeba
-HTML: Używaj znaczników semantycznych
-CSS: Grupuj style, używaj kebab-case
-JS: rozbijaj kod na moduły

## Współpraca
-przypisuj się do zadań zanim zaczniesz nad nimi pracę
